回文 串是向前和向后读都相同的字符串。


### （1）题目意思与生僻概念解释

**题目意思：**
给定一个字符串 `s`，要求将其像切蛋糕一样切成若干段，必须保证切出来的**每一段**都是一个“回文串”。需要找出所有满足这种条件的切分方案，并以列表的形式返回。

**核心概念解释：**
* **回文串（Palindrome）：** 正着读和反着读完全一样的字符串。例如 `"a"`、`"aa"`、`"aba"` 都是回文串，而 `"ab"` 不是。
* **子串（Substring）：** 字符串中连续的一段字符。
* **回溯算法（Backtracking）：** 本质是穷举搜索。在切分字符串时，先尝试在第一个字符后切一刀，如果切出来的前半段是回文，就基于这个选择，继续去切剩下的后半段。如果某次切出来的前半段不是回文，说明这条路走不通，直接放弃（剪枝），然后尝试在第二个字符后切一刀，以此类推。

---

### （2）代码解题思路解析

这段代码巧妙地使用了**“前缀判断 + 递归后缀”**的回溯模板，思路拆解如下：

1. **定义检验工具：** 首先需要一个能够快速判断字符串是否为回文串的工具。
2. **制定切分策略（找前缀）：** 面对一个待处理的字符串 `s`，使用循环枚举第一刀的切分位置。切分点可以从第 1 个字符一直到字符串的最后 1 个字符。每次切分都会产生一个“前缀”和一个“后缀”。
3. **验证与深入：**
    * **验证前缀：** 检查刚刚切下来的“前缀”是不是回文串。
    * **如果不是：** 说明这种切法无效，直接跳过，尝试把切分点往后移。
    * **如果是：** 说明当前切法有戏。将这个前缀记录到当前的“切割路径（`path`）”中，然后带着这条路径，**递归调用函数自身，去处理剩下的“后缀”**。
4. **成功条件（触底反弹）：** 随着不断递归，后缀越来越短。当某次递归传入的字符串变成空串（`not s`）时，说明原字符串已经被从头到尾完美地切分完毕了。此时，将积累下来的 `path` 装入总结果集 `res` 中。

### （3）带有注释的代码框

```python
class Solution(object):
    def partition(self, s):
        # 定义一个匿名函数(lambda)用于判断字符串是否为回文
        # 原理：将字符串反转(s[::-1])并与原字符串比较
        self.isPalindrome = lambda s : s == s[::-1]
        # 初始化全局结果列表，存放所有成功的切分方案
        res = []
        # 触发回溯搜索：传入完整的初始字符串，结果引用，以及初始为空的路径列表
        self.backtrack(s, res, [])
        # 搜索结束，返回最终收集到的所有方案
        return res
    def backtrack(self, s, res, path):
        # 终止条件：如果待处理的字符串为空，说明之前的切分全部合法且已切完
        if not s:
            # 将当前这条成功的切分路径加入总结果中
            res.append(path)
            return
        # 遍历所有可能的切分位置，i 代表切下的前缀长度
        # 长度从 1 开始，最多可以等于当前字符串的总长度
        for i in range(1, len(s) + 1): 
            # 截取前缀 s[:i]，并利用预先定义的函数判断其是否为回文
            if self.isPalindrome(s[:i]):
                # 若前缀是回文，则将其加入当前路径，并对剩余的后缀 s[i:] 继续进行递归回溯
                # 注意：path + [s[:i]] 会隐式创建一个新列表，避免了传统回溯中繁琐的 append 和 pop 操作
                self.backtrack(s[i:], res, path + [s[:i]])
```

### 1. 核心魔法：`s[::-1]` （字符串反转）
这是 Python 的**切片（Slicing）**用法。
切片的标准语法是 `[开始位置 : 结束位置 : 步长]`。
* 它的物理意义就是：**从字符串的最后一位开始，倒着走，把所有字符取一遍。**
* **算例：** 如果 `s = "abc"`，那么 `s[::-1]` 的结果直接就是 `"cba"`。

### 2. 判断逻辑：`s == s[::-1]`
回文串的定义就是“正着读反着读都一样”。
这行逻辑就是拿原字符串 `s`，和翻转后的字符串 `s[::-1]` 进行比较。
* 如果 `s = "aba"`，翻转后还是 `"aba"`。`"aba" == "aba"` 成立，返回 **`True`**。
* 如果 `s = "abc"`，翻转后是 `"cba"`。`"abc" == "cba"` 不成立，返回 **`False`**。

### 3. 外层包装：`lambda s :` （匿名函数）
`lambda` 是 Python 里用来定义**“匿名函数”**（也就是没有正经名字的微型函数）的关键字。
* 这里的 `s` 就是这个微型函数接收的参数。
* 冒号 `:` 后面跟着的就是这个函数要计算并返回的表达式。
* 连起来 `lambda s : s == s[::-1]` 的意思就是：
制造一个临时的处理机器，只要丢给它一个字符串 `s`，它就会吐出一个 `True` 或 `False` 的结果。
最后，把刚刚制造的这台“临时处理机器”，赋值给 `self.isPalindrome`。
这样，这个类里面的其他方法（比如后面的 `self.backtrack`），随时都可以通过调用 `self.isPalindrome(要检查的字符串)` 来鉴定回文串了。

这行看似高深的代码，如果**不使用 `lambda`**，
用最熟悉的基础语法写出来，完全等价于下面这个标准的函数定义：

        return True
    else:
        return False

4.  `res = []`

    * **解释：** 说明这是一次完美的彻头彻尾的切分。将一直携带来的合法路径 `path` 直接追加到全局结果 `res` 里。

    * **解释：** 成功收获一种方案后，结束当前这一层递归，返回上一层去寻找其他切分可能。
11. `for i in range(1, len(s) + 1):`
    * **解释：** 开启一个循环，尝试寻找下刀的位置。`i` 代表切下来的“前缀”的长度。因为前缀至少要有 1 个字符，最多可以包含整个当前字符串，所以范围是 `1` 到 `len(s)`。（`range` 右侧开区间，故写成 `len(s) + 1`）。
12. `if self.isPalindrome(s[:i]):`
    * **解释：** 利用 Python 的切片 `s[:i]` 截取出从开头到第 `i` 个字符组成的前缀。然后调用之前定义的 `isPalindrome` 判断它是不是回文串。如果不是，直接略过进入下一次循环尝试更长的前缀。
13. `self.backtrack(s[i:], res, path + [s[:i]])`
    * **解释：** 核心的深入探索步骤。如果前缀是回文：
        * 将剩余的后缀 `s[i:]` 作为下一次待处理的字符串传入。
        * 全局 `res` 照常传入。
        * 将当前合法的前缀变成单元素列表 `[s[:i]]`，并与历史路径 `path` 拼接。在 Python 中，列表的 `+` 运算会**生成一个全新的列表对象**传给下一层。这是一种非常优雅的做法，天然地处理了回溯算法中状态还原的问题，省去了显式执行 `path.pop()` 的麻烦。
